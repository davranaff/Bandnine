# Страницы фронтенда по ролям

## Общие страницы (без авторизации)

### Регистрация (`/register`)
- Поля: `store_name`, `email`, `password`
- Результат: создание магазина (tenant) + admin-аккаунта + JWT токен
- Редирект → POS

### Логин (`/login`)
- Поля: `email`, `password`
- Результат: JWT токен
- Редирект → POS

---

## Страницы после логина

### 1. POS — Создание продажи (`/pos`)
Центральный экран системы. Оптимизирован под скорость и минимум кликов.

**Элементы:**
- Поиск товаров (instant search)
- Добавление товара в корзину (1 click)
- Корзина (правая панель, sticky)
- Редактирование количества в корзине
- Удаление товара из корзины
- Real-time подсчёт итоговой суммы
- Выбор клиента (или guest)
- Выбор типа оплаты: cash / card / debt
- Кнопка "Завершить продажу"

---

### 2. Товары (`/products`)

#### 2.1 Список товаров (`/products`)
- Таблица: name, sku, stock, purchase_price, sale_price
- Поиск по name и sku
- Пагинация
- Кнопка "Создать товар"
- Row actions: Edit, Edit Stock

#### 2.2 Создание товара (модалка)
- Поля: name, sku (опционально), purchase_price, sale_price, initial_stock
- Валидация: name не пустой, цены >= 0, stock >= 0

#### 2.3 Редактирование товара (модалка)
- Поля: name, sku, purchase_price, sale_price
- Stock НЕ редактируется здесь

#### 2.4 Изменение остатка (модалка)
- Режим: "set" (установить значение) или "increment" (добавить/убавить)
- Валидация: stock >= 0

---

### 3. Клиенты (`/clients`)

#### 3.1 Список клиентов (`/clients`)
- Таблица: name, phone, created_at
- Поиск по имени и телефону
- Пагинация
- Кнопка "Добавить клиента"

#### 3.2 Создание клиента (модалка)
- Поля: name, phone
- Валидация: оба поля обязательны, name ≤ 255, phone ≤ 20

#### 3.3 Карточка клиента (`/clients/:id`)
- Данные клиента: name, phone
- История покупок: список продаж (sale_id, total, date)

---

### 4. История продаж (`/sales`)

#### 4.1 Список продаж (`/sales`)
- Таблица: sale_id, client_name (или "Guest"), total_amount, payment_type, created_at
- Пагинация

#### 4.2 Детали продажи (`/sales/:id` или модалка)
- Информация: клиент, дата, тип оплаты, итоговая сумма
- Список товаров в продаже: product_name, quantity, price

---

### 5. Долги (`/debts`)

#### 5.1 Список долгов (`/debts`)
- Таблица: client_name, total_amount, paid_amount, remaining, status
- Фильтр по статусу: active / closed / all
- Пагинация

#### 5.2 Детали долга (`/debts/:id` или модалка)
- Информация: клиент, общая сумма, оплачено, остаток, статус
- История платежей: amount, created_at
- Кнопка "Добавить платёж"

#### 5.3 Добавление платежа (модалка)
- Поле: amount
- Валидация: amount > 0, amount ≤ remaining
- Автозакрытие долга при полной оплате

## Общие UI-компоненты

### Layout
- **Sidebar** — навигация (отображает только доступные разделы по роли)
- **Top bar** — название магазина, имя пользователя, роль
- **Main content area** — основной контент

### Паттерны UX
- Модалки для создания/редактирования (без перехода на отдельную страницу)
- Toast-уведомления (успех/ошибка)
- Loading states (скелетоны)
- Sortable / searchable таблицы
- Пагинация на всех списках

## Примечания

- Seller после логина попадает сразу на POS и видит только его
- Admin/Manager после логина попадают на POS, но могут переключаться между разделами
- Регистрация всегда создаёт нового tenant + admin. Invite-механизм для добавления manager/seller в существующий магазин в MVP не реализован
